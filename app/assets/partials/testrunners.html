Test Runners...what are they?

A test runner is an application that executes these unit-tests. It can also execute integration or acceptance tests. Tests are usually written in suites so that the test runner can execute just one suite of tests at a time so you can focus on only the tests that you care about. How often you run all tests can vary as much as every time you commit or when leading up to a release.

In Javascript, two test runners have become popular; Jasmine and Mocha.The basic Describe and it BDD pattern between the two frameworks is identical. Assertions are done differently as well as the interface to asynchronous tests are different but mostly they are comparable. Illuminate is using Mocha and Chai. Chai is a plugin to Mocha that allows you to use assertions. 1

A typical assertion unit-test in Illuminate using Mocha and Chai will look something like this..

expect(view.$('svg')).to.have.length(1);
expect(didInsertElementCalled).to.be.true;
