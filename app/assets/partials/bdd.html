BDD - Behavior Driven Development

Behavior driven development is a type of development, not to be confused with TDD which is Test Driven Development. BDD specifies that tests of any unit of software should be specified in terms of the desired behavior of the unit.1In other words, instead of defining a whole bunch of functions to perform your unit tests, you write a lot of scenarios that describe the context in which the test is being written in. BDD

describe('Campaigns Controller', function () {
    describe('#filteredCampaigns', function () {
       it('should have a length of 3 campaigns when loading data and applying no filtering', function() {
            var controller = CampaignsController.create();
            Em.run(function() {
               controller.set('content', data);
            });
            expect(controller.get('filteredCampaigns')).to.have.length(3);
            controller.destroy();
        });
   });
});

What if you want to run a multitude of tests that require the same initialization function, like loading data into the controller, Mocha supports a before() function as well as anafter()function which will happen after all the tests in that suite are executed. There is also a beforeEach()and an afterEach()to handle any code that you want to execute before and after each test.
